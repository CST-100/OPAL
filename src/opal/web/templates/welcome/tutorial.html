WHAT IS OPAL?

OPAL (Operations, Procedures, Assets, Logistics) is a local-first ERP system for small teams building hardware.

DATABASESingle SQLite file on your machine — no cloud, no servers
NETWORKAccessible to other machines on your local network
AUDIT TRAILEvery create, update, and delete is logged with who/when/what
TRACEABILITYEvery physical item gets a unique OPAL number for full chain-of-custody

PARTS & INVENTORY

PARTSDefine what you build with — each part has a unique system ID and configurable part number (e.g., KST-F-0001)
OPAL NUMBERSEach physical item or batch gets a unique OPAL-XXXXX number for traceability
TIERSClassify parts by criticality — e.g., Flight (full traceability), Ground (support equipment), Dev (prototypes)
TRACKINGSerialized parts get one OPAL number per unit; Bulk parts get one per batch/box
INVENTORYTrack quantity, location, lot number, expiration date, and calibration status per OPAL number
BOMDefine bill-of-materials: which components go into an assembly, and in what quantity

Find parts under INVENTORY → PARTS in the top nav.

PROCEDURES & EXECUTIONS

TEMPLATESStep-by-step work instructions — write once, execute many times. Found under MANUFACTURING → PROCEDURES.
VERSIONINGPublishing a template creates an immutable snapshot. Editing the template never changes published versions.
EXECUTIONSRunning a published version. Each execution captures data, tracks time, and consumes inventory. Found under MANUFACTURING → EXECUTIONS.
WORK ORDERSExecutions get a WO-XXXXX number to group related work.
TYPESOP = operational procedure (no output). BUILD = produces an assembly (creates new OPAL-numbered output).
KITTINGTemplates define which parts and quantities are needed. Inventory is consumed during execution.

ISSUES & RISKS

ISSUE TYPES Non-Conformance — hardware doesn't meet spec (has should-be / is fields)
Bug — software/system defect (has steps to reproduce)
Task — general work item
Improvement — feature request or enhancement
AUTO-CREATEDNon-conformances can be logged during procedure execution and automatically create linked issues
LINKINGIssues link to parts, procedures, and executions for full traceability
RISKSTrack project risks with probability × impact scoring (1-5 each). Score determines severity: low (≤5), medium (6-12), high (13-25).

Find issues and risks under QUALITY in the top nav.

NAVIGATION

TOP NAVThree dropdown menus: INVENTORY (parts, stock, purchases), MANUFACTURING (procedures, executions), QUALITY (issues, risks)
CMD PALETTECtrl+K — search anything, navigate anywhere, create new items. Fastest way to get around.
SHORTCUTSAlt+P (parts), Alt+R (procedures), Alt+E (executions), Alt+I (issues), Alt+K (risks), Alt+H (home)
DOCSFull documentation available in-app — find it in the user menu (top right).

The UI is dense and explicit by design — data tables over cards, no hidden information, monospace for structured data. Everything is visible, nothing is behind a menu you don't know about.
