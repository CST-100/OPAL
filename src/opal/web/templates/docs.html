OVERVIEW

OPAL (Operations, Procedures, Assets, Logistics) is an enterprise resource planning system optimized for small teams and hardware projects.

Key design principles:

  • Local-first - Runs on a single machine, SQLite database
  • Full traceability - Every part movement is tracked
  • Procedure versioning - Immutable snapshots when published
  • Dense UI - Show all relevant data, no hiding information

PARTS & INVENTORY

Parts

Parts are the core items in your inventory. Each part has:

  • ID - Unique, auto-incrementing, never reused
  • External PN - Manufacturer or supplier part number
  • Tier - Classification level (e.g., Flight, Ground, Loose)
  • Parent - Optional parent assembly for BOM structure
  • Category - Organizational grouping

Tiered Inventory

Parts are classified into tiers defined in your project config:

  • Tier 1 (Flight) - Flight-critical hardware requiring full traceability
  • Tier 2 (Ground) - Ground support equipment
  • Tier 3 (Loose) - Consumables and non-critical items

Tier codes are embedded in part numbers (e.g., RV1-F-0001 for Flight tier).

OPAL Numbers

Each physical inventory item gets a unique OPAL number (OPAL-XXXXX) for full traceability. This tracks:

  • Source (purchase order, production, manual entry)
  • Location and movements
  • Consumption in procedures
  • Lot/batch information

Bill of Materials (BOM)

Parts can be organized into assemblies:

  • Set a part's Parent to create hierarchy
  • Use BOM Lines to define quantities of components in an assembly
  • View the assembly tree from any part's detail page

PROCEDURES

Master Procedures

Procedures define step-by-step processes. Key concepts:

  • Draft/Active/Deprecated - Lifecycle states
  • Steps - Individual actions with instructions (supports Markdown)
  • Sub-steps - Nested steps for complex operations
  • Contingency steps - Only shown when a non-conformance occurs
  • Kitting - Required parts/quantities for the procedure

Versioning

When you Publish a procedure:

  • Creates an immutable snapshot (version)
  • Future edits don't affect published versions
  • Executions always reference a specific version

Executions

An execution is a run-through of a published procedure:

  • Tracks status of each step (pending, in progress, completed, skipped)
  • Captures data defined in step schemas
  • Records time spent on each step
  • Logs non-conformances (auto-creates Issues)
  • Consumes inventory items from kit

ISSUES & RISKS

Issues

Track problems and tasks across your project:

  • Types: non_conformance, bug, task, improvement
  • Priority: low, medium, high, critical
  • Status: open, in_progress, resolved, closed
  • Can link to parts, procedures, or executions
  • Auto-created from non-conformances during execution

Risks

Proactive risk management with probability x impact scoring:

  • Probability: 1-5 scale
  • Impact: 1-5 scale
  • Severity: Calculated as P x I (low/medium/high)
  • View risks on the 5x5 matrix at /risks/matrix

PROJECT CONFIGURATION

Each project has an opal.project.yaml file defining:

  • Tiers - Inventory classification levels with codes
  • Part Numbering - Prefix, separator, format template
  • Requirements - Project-level requirements assignable to parts
  • Categories - Part category suggestions

Requirements

Define requirements in your project config, then:

  • Assign requirements to parts
  • Track status: open, verified, waived, not_applicable
  • Record verification notes and timestamps

Running with a Project

# From OPAL source directory:
uv run opal serve --project /path/to/your/project

# Initialize a new project's database:
uv run opal init --project /path/to/your/project

MCP / CLAUDE CODE INTEGRATION

OPAL includes an MCP (Model Context Protocol) server for integration with Claude Code.

Setup

Add to your Claude Code config (~/.claude.json):

{
  "mcpServers": {
    "opal": {
      "command": "uv",
      "args": ["run", "opal", "mcp", "--project", "/path/to/project"],
      "cwd": "/path/to/opal/source"
    }
  }
}

Available Tools

  • get_project_info - Get project config, tiers, requirements
  • create_part - Create a new part
  • list_parts - Search and list parts
  • get_part - Get part details
  • preview_part_number - Preview part number format
  • list_requirements - List project requirements
  • assign_requirement - Assign requirement to part
  • verify_requirement - Mark requirement as verified
  • get_bom - Get bill of materials for assembly
  • add_component - Add component to assembly BOM
